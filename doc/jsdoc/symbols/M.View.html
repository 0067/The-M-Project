Field Detail
childViews
An array specifying the view's children.
contentBinding
The path to a content that is bind to the view's value.
cssClass
This property can be used to assign a css class to the view using the jquery mobile specific data-theme property. This allows you to create your custom styles.
cssClassOnError
This property can be used to assign a css class to the view if an error occurs. The applying of this class is automatically triggered if the validation of the view goes wrong.
cssClassOnInit
This property can be used to assign a css class to the view on its initialization. This property is mainly used for input ui elements like text fields, that might have a initial value that should be rendered in a different style than the later value entered by the user.
cssStyle
This property can be used to assign a css style to the view using. This allows you to create your custom styles inline.
hasFocus
Indicates whether this view currently has the focus or not.
html
This property is used internally to recursively build the pages html representation. It is once set within the render method and then eventually updated within the renderUpdate method.
id
The id of the View used for the HTML attribute ID.
isInline
Indicates whether the view should be displayed inline or not.
isView
A boolean value to definitely recognize a view as a view.
isVisible
Indicates whether the view is visible or not. Implemented via CSS's display property.
parentView
This property can be used to save a reference to the view's parent view.
renderToDOM
Determines whether this view renders directly to the DOM or just returns its HTML representation.
triggerActionOnChange
Determines whether an onChange event will trigger a defined action or not. This property is basically interesting for input ui elements, e.g. for text fields.
triggerActionOnEnter
Determines whether an onKeyUp event with the enter button will trigger a defined action or not. This property is basically interesting for input ui elements, e.g. for text fields.
triggerActionOnKeyUp
Determines whether an onKeyUp event will trigger a defined action or not. This property is basically interesting for input ui elements, e.g. for text fields.
type
The type of this object.
value
Generic attribute for views. Some subclasses have a property that is more readable to its context than 'value'. Internally these properties are mapped to 'value'. e.g. in LabelView: value:this.text
